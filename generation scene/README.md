# InfiniteWorld

A 3D scene auto-generation and Isaac Sim import tool based on [Holodeck](https://github.com/allenai/Holodeck). This project contains two core scripts:

- `main_gen.py`: Calls a Large Language Model (LLM) to automatically generate 3D indoor scene JSON files from text descriptions.
- `multi_place.py`: Batch-imports generated scene JSON files into NVIDIA Isaac Sim to build simulatable USD scenes.

---

## Usage

### `main_gen.py` — Scene Generation

#### Single Scene Generation

Generate a single 3D indoor scene from a text description:

```bash
python main_gen.py \
  --mode generate_single_scene \
  --query "a cozy bedroom with a bookshelf" \
  --openai_api_key YOUR_API_KEY \
  --save_dir /path/to/output/
```

#### Batch Scene Generation

Read descriptions line by line from a text file and generate multiple scenes in batch:

```bash
python main_gen.py \
  --mode generate_multi_scenes \
  --query_file /path/to/queries.txt \
  --openai_api_key YOUR_API_KEY \
  --save_dir /path/to/output/
```

Each line in `queries.txt` is one scene description, for example:

```
a modern living room
a kitchen with an island
a small home office
```

#### Scene Variant Generation

Generate a number of variants based on an existing scene JSON:

```bash
python main_gen.py \
  --mode generate_variants \
  --query "a cozy bedroom" \
  --original_scene /path/to/scene.json \
  --number_of_variants 5 \
  --openai_api_key YOUR_API_KEY \
  --save_dir /path/to/output/
```

### `multi_place.py` — Isaac Sim Scene Import

> ⚠️ This script must be run inside the **NVIDIA Isaac Sim** built-in Python environment. It does not support a standalone Python interpreter.

#### Overview

The script traverses all scene JSON files in the specified input folder, converts each scene into an Isaac Sim USD file, and saves the result to the output directory. The conversion handles:

- **Floor objects**: Large furniture, placed directly by coordinates.
- **Small objects**: Small items, with the correct vertical position calculated automatically based on object height.
- **Walls**: Wall geometry generated from polygon vertices, with door openings handled automatically.
- **Doors**: Openings cut into walls; each wall is split into three segments around the gap.
- **Room floors**: Floor planes generated from room vertices.

#### Coordinate System Conversion

The script includes built-in conversion logic from the AI2-THOR coordinate system to the Isaac Sim coordinate system:

- Position: `(x, y, z)` → `(x, z, 0)` (different vertical axis)
- Rotation: AI2-THOR Euler angles are aligned to the Isaac Sim coordinate system by applying an additional 90° rotation around the X axis.

#### Configuration

Before running, update the following path variables in the script to match your local environment:

```python
# Input folder for scene JSON files
input_folder_path = '/path/to/scene/input/'

# Output folder for generated USD files
output_folder_path = '/path/to/scene/output/'

# Objaverse USD asset root directory (floor_objects / small_objects)
path = "/path/to/objaverse_holodeck/usd/" + assetId + "/raw_model3.usd"

# Floor USD model path (shared by walls and floors)
floor_path = "/path/to/floor.usd"
```

#### Running

Run via the Isaac Sim built-in Python:

```bash
# Adjust the path to match your Isaac Sim installation
~/.local/share/ov/pkg/isaac_sim-*/python.sh multi_place.py
```

## Data Format

The scene JSON files generated by `main_gen.py` follow the structure below (read by `multi_place.py`):

```json
{
  "rooms": [
    { "vertices": [{"x": 0, "y": 0}, ...] }
  ],
  "walls": [
    {
      "id": "wall_0",
      "polygon": [{"x": 0, "y": 0, "z": 0}, ...]
    }
  ],
  "doors": [
    {
      "wall0": "wall_0",
      "wall1": "wall_1",
      "holePolygon": [{"x": 0.5, "y": 0}, {"x": 1.5, "y": 2.1}]
    }
  ],
  "floor_objects": [
    {
      "assetId": "obj_001",
      "position": {"x": 1.0, "y": 0.0, "z": 2.0},
      "rotation": {"x": 0, "y": 90, "z": 0}
    }
  ],
  "small_objects": [
    {
      "assetId": "obj_002",
      "position": {"x": 1.0, "y": 0.8, "z": 2.0},
      "rotation": {"x": 0, "y": 0, "z": 0},
      "height": 0.2
    }
  ]
}
```

## Notes

1. **OpenAI API Key**: `main_gen.py` requires a valid OpenAI API key. Pass it via `--openai_api_key` or configure it inside `SceneGenerator`.
2. **Asset Paths**: The USD asset paths in `multi_place.py` are currently hardcoded. Update them manually when deploying on a different machine.
3. **Isaac Sim Version Compatibility**: `multi_place.py` uses the `omni.isaac.core` API. API differences may exist across Isaac Sim versions — refer to the documentation for your specific version.
4. **Scene Reuse**: When generating a single scene, `main_gen.py` will automatically load and continue from an existing scene JSON at the target path, rather than generating from scratch.
